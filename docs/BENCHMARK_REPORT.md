## Expected Outcomes

### `file-descriptor-limit-fixture`

- The skill should compare open descriptor counts with the soft nofile limit before recommending policy changes.
- The skill should preserve workflow fan-out and many-small-file context as evidence.

### `gpu-sanity-static`

- The skill should verify scheduler allocation and runtime visibility before framework-specific debugging.
- The skill should keep GPU checks small enough for a short reviewed allocation.

### `ior-mdtest-storage-static`

- The skill should require local policy review before running storage benchmarks.
- The skill should frame results as support triage evidence, not broad site performance claims.

### `job-failure-triage-static`

- The skill should collect evidence before recommending a fix.
- The skill should route common failure classes such as OOM, timeout, modules, permissions, and node failure.

### `mpi-hello-site`

- The site smoke test should only run after local account, partition, and MPI module values are supplied out of band.
- The submitted job should compile hello_mpi.c and finish within the benchmark timeout on a reviewed Slurm site.

### `mpi-hello-static`

- The skill should provide a tiny MPI sanity check that can be reviewed before running on a real allocation.
- The skill should preserve launcher and module assumptions for local site review.

### `quota-filesystem-fixture`

- The skill should distinguish filesystem fullness, quota, inode exhaustion, and path permission evidence.
- The skill should ask for redacted path and log context before suggesting site-specific remediation.

### `shared-permissions-fixture`

- The skill should treat traversal, group membership, setgid inheritance, and ACL masks as separate evidence paths.
- The skill should avoid suggesting chmod, chgrp, chown, or setfacl changes without local support review.

### `slurm-node-failure-fixture`

- The skill should separate application failure evidence from infrastructure or launch-path evidence.
- The skill should recommend support handoff or scoped retry review before repeated large resubmissions.

### `slurm-oom-fixture`

- The skill should ask the user to compare ReqMem, MaxRSS, job steps, and logs before increasing memory.
- The skill should treat missing or incomplete accounting as possible and preserve application-level evidence.

### `slurm-output-log-fixture`

- The skill should inspect scheduler log paths, working directory, and filesystem state before blaming Slurm.
- The skill should route existing but empty logs toward job-state and application-startup review.

### `slurm-pending-reason-fixture`

- The skill should identify dependency waits separately from priority, resources, reservation, and unavailable-node waits.
- The skill should treat scheduler priority values as policy signals, not start-time promises.

### `slurm-qos-account-limit-fixture`

- The skill should map AssocGrp and QOSGrp reasons to aggregate account or QOS limits.
- The skill should compare queue reason, account, QOS, fairshare, and visible TRES policy before suggesting job changes.

### `slurm-submit-static`

- Examples are parseable as shell scripts without submitting jobs.
- Scheduler account and partition remain placeholders until a site or user supplies values.

### `slurm-time-limit-fixture`

- The skill should distinguish normal progress at time limit from immediate application failure.
- The skill should suggest checkpointing, splitting, or requesting more time only after reviewing progress evidence.
